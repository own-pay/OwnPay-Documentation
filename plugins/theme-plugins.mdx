---
title: "Theme plugins"
description: "Overview of OwnPay theme plugins that override checkout templates, styles, and layouts to match your brand look and feel without touching core code."
---

Theme plugins customize the visual appearance of OwnPay's checkout pages and customer-facing interfaces.

## Theme structure

```
modules/themes/my-theme/
+-- manifest.json
+-- templates/
|   +-- checkout/
|       +-- checkout.twig
|       +-- success.twig
|       +-- failure.twig
+-- assets/
|   +-- css/
|   |   +-- checkout.css
|   +-- js/
|       +-- checkout.js
+-- src/
    +-- ThemePlugin.php
```

## manifest.json

```json
{
  "slug": "my-theme",
  "name": "My Custom Theme",
  "version": "1.0.0",
  "type": "theme",
  "entrypoint": "src/ThemePlugin.php",
  "namespace": "OwnPay\\Themes\\MyTheme"
}
```

## Twig templates

Override default templates by placing them in your theme's `templates/` directory:

### checkout.twig

```twig
<!DOCTYPE html>
<html>
<head>
    <title>Checkout - {{ brand.name }}</title>
    <link rel="stylesheet" href="/assets/css/checkout.css">
    {{ hook('checkout.head') }}
</head>
<body>
    <div class="checkout-container">
        <header>
            {% if brand.logo %}
                <img src="{{ brand.logo }}" alt="{{ brand.name }}">
            {% endif %}
        </header>
        
        <main>
            <h1>Complete Your Payment</h1>
            
            <form method="POST" action="/checkout/{{ transaction.id }}/process">
                {{ csrf_field() }}
                
                <div class="amount">
                    {{ transaction.amount|currency(transaction.currency) }}
                </div>
                
                <div class="payment-methods">
                    {% for gateway in gateways %}
                        <label class="payment-method">
                            <input type="radio" name="gateway" value="{{ gateway.slug }}">
                            <img src="{{ gateway.icon }}" alt="{{ gateway.name }}">
                            {{ gateway.name }}
                        </label>
                    {% endfor %}
                </div>
                
                <button type="submit" class="pay-button">
                    Pay {{ transaction.amount|currency(transaction.currency) }}
                </button>
            </form>
        </main>
        
        {% if brand.show_powered_by %}
            <footer>
                Powered by OwnPay
            </footer>
        {% endif %}
    </div>
    
    {{ hook('checkout.footer') }}
</body>
</html>
```

## Custom CSS

Brand-specific CSS is injected automatically:

```css
:root {
    --teal: {{ brand.color }};
    --teal-deep: {{ brand.accent_color }};
}

.checkout-container {
    font-family: 'Inter', sans-serif;
    max-width: 480px;
    margin: 0 auto;
    padding: 2rem;
}
```

## Brand theme variables

Access brand customization in templates:

| Variable | Description |
|----------|-------------|
| `brand.name` | Brand display name |
| `brand.logo` | Brand logo URL |
| `brand.favicon` | Brand favicon URL |
| `brand.color` | Primary color |
| `brand.accent_color` | Accent color |
| `brand.custom_css` | Custom CSS |
| `brand.custom_js` | Custom JavaScript |
| `brand.footer_text` | Custom footer text |
| `brand.show_powered_by` | Show "Powered by OwnPay" |

## Asset management

Place static assets in `assets/`:

- `assets/css/` - Stylesheets
- `assets/js/` - JavaScript files
- `assets/images/` - Images

Reference them in templates:

```twig
<link rel="stylesheet" href="/modules/themes/my-theme/assets/css/checkout.css">
<script src="/modules/themes/my-theme/assets/js/checkout.js"></script>
```

## Theme activation

1. Go to **System > Themes**
2. Find your theme in the list
3. Click **Activate**
4. The theme applies to all checkout pages

## Best practices

- Keep checkout simple and focused
- Optimize for mobile devices
- Test with screen readers for accessibility
- Use brand colors consistently
- Minimize external dependencies
- Ensure fast page load times
