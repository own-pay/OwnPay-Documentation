This guide walks you through creating and processing your first payment with OwnPay.

## Prerequisites

- OwnPay installed and running
- At least one payment gateway configured
- A brand created with gateway enabled

## Step 1: Create a payment link

1. Log in to your OwnPay admin panel
2. Go to **Payments > Payment Links**
3. Click **Create Payment Link**
4. Fill in:
   - **Amount** - Payment amount (e.g., 50.00)
   - **Currency** - Select currency (e.g., USD)
   - **Description** - What the payment is for
   - **Customer Email** - Optional, for pre-filling
5. Click **Create**

## Step 2: Test with sandbox

1. Go to **Gateways**
2. Ensure your gateway is in **Test Mode**
3. Use test card numbers:

| Card Type | Number | Outcome |
|-----------|--------|---------|
| Visa | 4111 1111 1111 1111 | Success |
| Mastercard | 5555 5555 5555 4444 | Success |
| Amex | 3782 822463 10005 | Success |
| Declined | 4000 0000 0000 0002 | Failure |

## Step 3: Complete the payment

1. Open the payment link URL in a browser
2. Enter test card details
3. Click **Pay**
4. You should see a success page

## Step 4: Verify the transaction

1. Go to **Payments > Transactions**
2. Find your test payment
3. Verify status shows **Completed**

## Step 5: Switch to live mode

Once testing is complete:

1. Go to **Gateways > [Your Gateway]**
2. Switch from **Test Mode** to **Live Mode**
3. Enter your live API credentials
4. Test with a real (small) transaction

## Via API

You can also create payments programmatically:

```bash
curl -X POST https://yourdomain.com/api/v1/payments \
  -H "Authorization: Bearer op_live_xxx..." \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 5000,
    "currency": "USD",
    "customer_email": "customer@example.com",
    "description": "Order #123"
  }'
```

The response includes a `checkout_url` where your customer makes the payment.
