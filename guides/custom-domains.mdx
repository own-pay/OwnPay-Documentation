---
title: "Set up custom domains"
description: "Step-by-step guide to point a white-label domain at a brand, verify DNS records, and provision SSL certificates for checkout pages."
---

This guide covers configuring custom domains for brands so customers never see the master OwnPay domain.

## Prerequisites

- A brand created in OwnPay
- A domain name you control
- DNS access to configure records

## Step 1: Add the domain

1. Go to **System > Domains**
2. Click **Add Domain**
3. Enter the domain (e.g., `pay.yourdomain.com`)
4. Select domain type: **checkout**
5. Click **Save**

## Step 2: Configure DNS

Add one of these DNS records:

### Option A: A Record

| Type | Name | Value | TTL |
|------|------|-------|-----|
| A | pay | YOUR_SERVER_IP | 300 |

### Option B: CNAME Record

| Type | Name | Value | TTL |
|------|------|-------|-----|
| CNAME | pay | yourdomain.com | 300 |

## Step 3: Verify DNS

1. Go to **System > Domains**
2. Click **Verify DNS** next to your domain
3. Wait for verification (usually within minutes)

<Note>
The domain must have `dns_verified = 1` before OwnPay will serve content on it. Until verified, visitors see a 503 page.
</Note>

## Step 4: Install SSL

OwnPay requires HTTPS for custom domains. Install a free Let's Encrypt certificate:

```bash
sudo certbot --nginx -d pay.yourdomain.com
```

## Step 5: Test

1. Visit `https://pay.yourdomain.com/checkout/[payment-id]`
2. Verify the checkout page loads with the brand's customization
3. Complete a test payment

## Domain types

| Type | Serves | Use Case |
|------|--------|----------|
| `checkout` | Checkout, invoice, payment pages | Customer-facing payments |
| `api` | API endpoints only | Programmatic access |

## Security

- Admin routes (`/admin/*`) return 404 on all custom domains
- Custom domains require DNS verification before serving content
- Inactive or suspended brands return 404 on their custom domains

## Troubleshooting

| Issue | Solution |
|-------|----------|
| 503 "DNS verification pending" | Click Verify DNS in System > Domains |
| 404 on checkout pages | Ensure domain type is `checkout`, not `api` |
| SSL errors | Install a valid SSL certificate |
| Brand not loading | Check brand status is `active` in People > Brands |
