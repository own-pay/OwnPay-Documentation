This guide covers adding and configuring payment gateways in OwnPay.

## Adding a gateway

1. Go to **Gateways > Payment Gateways**
2. Click **Add Gateway**
3. Select the gateway type
4. Enter credentials:
   - API Key / Merchant ID
   - Secret Key
   - Account ID (if applicable)
5. Choose payment methods to enable
6. Click **Save**

## Gateway credentials

Each gateway needs:

| Credential | Description |
|-----------|-------------|
| **API Key** | Public identifier |
| **Secret Key** | Private authentication |
| **Merchant ID** | Account identifier |

<Warning>
Never share secret keys. Rotate them regularly.
</Warning>

## Test mode

1. Go to **Gateways > [Gateway Name]**
2. Toggle **Test Mode** on
3. Use test card numbers provided by the gateway
4. Verify payments process correctly
5. Switch to **Live Mode** when ready

## Per-brand configuration

Each brand configures its own gateway credentials:

1. Switch to the brand context
2. Go to **Gateways**
3. Configure the brand's gateway credentials
4. The brand's checkout uses these credentials

## Manual gateways

OwnPay supports manual (offline) gateways for methods like bank transfer:

1. **Template** - Platform-owned type definition (created in All Brands view)
2. **Account** - Per-brand configuration with brand-specific instructions

The template defines the gateway type; each brand adds its own account with payment instructions.

## Currency configuration

1. Go to **Gateways > Currencies**
2. Add currencies you accept
3. Set exchange rates (for automatic conversion)
4. Update rates periodically

## Best practices

- Use 2+ gateways for redundancy
- Test thoroughly in sandbox before going live
- Monitor success rates
- Rotate API keys quarterly
- Never log sensitive credentials
