OwnPay is a self-hosted, open-source payment gateway platform that gives you complete control over payment processing. It supports multi-brand management, 100+ payment gateways, white-label checkout experiences, and a full REST API for developers.

<Info>
**What is OwnPay?** OwnPay is infrastructure you deploy and own. You control the server, the database, the credentials, and the data. No third-party intermediary has access to your transactions.
</Info>

## Key features

| Feature | Description |
|---------|-------------|
| **Multi-brand Management** | Run multiple stores, brands, or merchants from a single installation with complete data isolation |
| **100+ Payment Gateways** | Connect to Stripe, PayPal, bKash, Nagad, GCash, bank transfers, and more |
| **White-label Checkout** | Custom domains, logos, CSS, and email templates for each brand |
| **Double-entry Ledger** | Financial integrity with double-entry bookkeeping for all transactions |
| **REST API** | Full programmatic access with API keys, webhooks, and SDK support |
| **Plugin System** | Extend with gateway plugins, addon plugins, and theme plugins |
| **Mobile & SMS** | Android companion app for SMS-based payment verification |
| **RBAC Permissions** | Role-based access control for administrators, brand managers, and staff |

## Quick start

<Steps>
<Step title="Install OwnPay">
  Upload files to your server and run the web installer. See [Installation Guide](/installation).
</Step>
<Step title="Create your first brand">
  Log in as administrator and create a brand. See [Brands](/user-guide/people/brands).
</Step>
<Step title="Configure a payment gateway">
  Connect a gateway like Stripe or bKash. See [Gateway Setup](/user-guide/gateways/gateways).
</Step>
<Step title="Start accepting payments">
  Create payment links or integrate via API. See [Payment Links](/user-guide/payments/payment-links).

</Steps>
