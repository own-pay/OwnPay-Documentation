OwnPay can automatically verify payments by parsing SMS notifications from mobile banking services.

## How SMS verification works

1. Customer initiates payment on checkout page
2. Customer pays via mobile banking (bKash, Nagad, etc.)
3. Banking SMS arrives on paired companion device
4. `SmartSmsAnalyzer` parses the SMS content
5. `SmsVerificationJob` matches SMS to pending transaction
6. Transaction status updates to completed
7. Ledger entry is posted

## SMS matching strategies

The system matches SMS to transactions in this order:

### 1. Provider transaction ID (exact match)

If the SMS contains a transaction ID that matches `op_transactions.provider_trx_id`, it's matched immediately.

### 2. Amount + gateway + time window (fallback)

If no transaction ID is parsed, the system looks for pending transactions matching:
- Same amount
- Same gateway
- Within the time window

<Warning>
The system refuses ambiguous matches. If two transactions have the same amount pending in the same window, neither is matched automatically. One must be manually completed.
</Warning>

## SMS templates

OwnPay includes built-in templates for common mobile banking formats. Add custom templates for new providers:

1. Go to **Mobile & SMS > SMS Templates**
2. Click **Add Template**
3. Define regex patterns for parsing:
   - Sender pattern
   - Amount pattern
   - Transaction ID pattern
   - Gateway identification

## Configuration

### Cron job

SMS verification runs via cron job:

```bash
* * * * * php /var/www/ownpay/public/index.php cron/run
```

### Manual trigger

```bash
php public/index.php cron/run
```

## Monitoring

### SMS logs

Go to **Mobile & SMS > SMS Logs** to view:
- Received SMS records
- Match status (pending, matched, failed)
- Parse results

### Pending SMS

Check for unmatched SMS:

```sql
SELECT id, device_id, amount, trx_id, gateway_slug, match_status
FROM op_sms_parsed
WHERE match_status = 'pending'
ORDER BY created_at DESC;
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| SMS not matching | Check SmartSmsAnalyzer can parse the format |
| Amount collision | Two same-amount transactions - manually complete one |
| Cron not running | Add cron job: `* * * * * php .../cron/run` |
| Wrong brand attribution | Check `op_sms_parsed.merchant_id` |
